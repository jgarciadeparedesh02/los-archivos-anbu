En el desarrollo de interfaces de usuario con Ionic, los **controles** (también conocidos como componentes UI) son los bloques fundamentales que permiten construir la apariencia y la interactividad de la aplicación. Ionic proporciona un amplio conjunto de componentes pre-construidos que siguen las directrices de diseño de iOS y Android, facilitando la creación de aplicaciones multiplataforma con una experiencia de usuario nativa.

Cada control de Ionic es un componente web que se utiliza de forma declarativa en las plantillas HTML de Angular, y sus características se personalizan a través de **propiedades** (atributos).

### **Tipos de Componentes Visuales en Ionic**

Ionic organiza sus componentes en categorías lógicas para facilitar su uso:

#### **1. Componentes de Estructura y Layout**

Estos componentes definen la disposición general de la aplicación y sus secciones principales.

*   **`<ion-app>`:** El componente raíz de toda aplicación Ionic.
*   **`<ion-header>`:** Contenedor para la barra de navegación superior, que a menudo incluye un título y botones de acción.
    *   **Ejemplo:**
        ```html
        <ion-header>
            <ion-toolbar>
                <ion-title>Mi Aplicación Ionic</ion-title>
            </ion-toolbar>
        </ion-header>
        ```
*   **`<ion-content>`:** El área principal donde se muestra el contenido de la página. Permite el desplazamiento y puede tener propiedades como `fullscreen` o `scroll-y`.
    *   **Ejemplo:**
        ```html
        <ion-content class="ion-padding">
            <!-- Contenido de la página aquí -->
            <p>Bienvenido a mi primera aplicación Ionic.</p>
        </ion-content>
        ```
*   **`<ion-footer>`:** Contenedor para la barra de navegación inferior o cualquier contenido que deba permanecer en la parte inferior de la pantalla.
    *   **Ejemplo:**
        ```html
        <ion-footer>
            <ion-toolbar>
                <ion-title>Pie de página</ion-title>
            </ion-toolbar>
        </ion-footer>
        ```
*   **`<ion-grid>`, `<ion-row>`, `<ion-col>`:** Componentes para crear layouts responsivos basados en un sistema de rejilla (grid system).

#### **2. Componentes de Entrada de Datos**

Permiten a los usuarios introducir información en la aplicación.

*   **`<ion-input>`:** Campo de entrada de texto para diferentes tipos de datos (texto, número, email, contraseña, etc.).
    *   **Propiedades comunes:** `type`, `placeholder`, `value`, `clear-input`, `debounce`.
    *   **Ejemplo:**
        ```html
        <ion-item>
            <ion-label position="floating">Correo Electrónico</ion-label>
            <ion-input type="email" placeholder="introduce@ejemplo.com"></ion-input>
        </ion-item>
        ```
*   **`<ion-textarea>`:** Área de texto para entradas multilínea.
*   **`<ion-checkbox>`:** Casilla de verificación.
*   **`<ion-radio>`:** Botón de opción (para selección única).
*   **`<ion-toggle>`:** Interruptor de encendido/apagado.
*   **`<ion-select>`:** Selector de opciones (dropdown).
*   **`<ion-datetime>`:** Selector de fecha y hora.

#### **3. Componentes de Navegación y Menús**

Facilitan la navegación entre diferentes partes de la aplicación.

*   **`<ion-tabs>`:** Componente para crear una interfaz de navegación basada en pestañas.
*   **`<ion-menu>`:** Menú lateral (drawer) que se desliza desde el borde de la pantalla.
*   **`<ion-router-outlet>`:** Donde Angular renderiza los componentes de las rutas.

#### **4. Componentes de Botones y Acciones**

Permiten a los usuarios interactuar con la aplicación y realizar acciones.

*   **`<ion-button>`:** Botón estándar con múltiples estilos y comportamientos.
    *   **Propiedades comunes:** `color`, `expand`, `fill`, `shape`, `size`, `disabled`.
    *   **Ejemplo:**
        ```html
        <ion-button color="primary" expand="block">
            <ion-icon slot="start" name="log-in"></ion-icon>
            Iniciar Sesión
        </ion-button>

        <ion-button color="secondary" fill="outline" shape="round">
            Registrarse
        </ion-button>
        ```
*   **`<ion-fab>`, `<ion-fab-button>`:** Botón de acción flotante (Floating Action Button), ideal para acciones primarias.

#### **5. Componentes de Visualización de Datos e Información**

Muestran información al usuario.

*   **`<ion-card>`:** Contenedor flexible para agrupar contenido relacionado.
*   **`<ion-list>`, `<ion-item>`:** Para mostrar listas de elementos.
*   **`<ion-avatar>`, `<ion-thumbnail>`, `<ion-img>`:** Para mostrar imágenes y avatares.
*   **`<ion-badge>`:** Pequeña etiqueta para mostrar contadores o estados.

### **Definición de Atributos Declarativos (Propiedades)**

Las propiedades de los componentes Ionic se definen directamente en la plantilla HTML como atributos. Ionic utiliza un sistema de propiedades consistente que permite personalizar la apariencia y el comportamiento de los componentes.

*   **`color`:** Define el color del componente (ej. `primary`, `secondary`, `tertiary`, `success`, `warning`, `danger`, `light`, `medium`, `dark`).
    *   `<ion-button color="danger">Eliminar</ion-button>`
*   **`expand`:** Controla el ancho del botón (ej. `block` para ocupar todo el ancho disponible).
    *   `<ion-button expand="full">Guardar</ion-button>`
*   **`fill`:** Define el estilo de fondo del botón (ej. `solid`, `outline`, `clear`).
    *   `<ion-button fill="outline">Editar</ion-button>`
*   **`mode`:** Permite especificar el modo de plataforma (iOS o Material Design) para un componente específico, anulando el modo global.
    *   `<ion-toggle mode="ios"></ion-toggle>`
*   **`shape`:** Define la forma del botón (ej. `round`).
    *   `<ion-button shape="round">Redondo</ion-button>`
*   **`slot`:** Se utiliza para posicionar contenido dentro de un componente padre (ej. `start`, `end` para iconos en botones).
    *   `<ion-icon slot="start" name="star"></ion-icon>`

### **Ejemplo de Personalización con Propiedades**

```html
<ion-header>
    <ion-toolbar color="primary">
        <ion-buttons slot="start">
            <ion-menu-button></ion-menu-button>
        </ion-buttons>
        <ion-title>Formulario de Contacto</ion-title>
    </ion-toolbar>
</ion-header>

<ion-content class="ion-padding">
    <ion-item>
        <ion-label position="stacked">Nombre Completo</ion-label>
        <ion-input type="text" placeholder="Introduce tu nombre"></ion-input>
    </ion-item>

    <ion-item>
        <ion-label position="floating">Email</ion-label>
        <ion-input type="email" placeholder="tu.email@ejemplo.com"></ion-input>
    </ion-item>

    <ion-item>
        <ion-label>Aceptar Términos</ion-label>
        <ion-checkbox color="secondary" slot="end"></ion-checkbox>
    </ion-item>

    <ion-button expand="block" color="success" class="ion-margin-top">
        <ion-icon slot="start" name="send"></ion-icon>
        Enviar Mensaje
    </ion-button>

    <ion-button expand="block" fill="clear" color="medium">
        Cancelar
    </ion-button>
</ion-content>
```

Este ejemplo demuestra cómo, utilizando diferentes componentes y sus propiedades, podemos construir una interfaz funcional y visualmente atractiva de manera declarativa en Ionic. La clave reside en conocer los componentes disponibles y las propiedades que ofrecen para personalizarlos según las necesidades del diseño.
