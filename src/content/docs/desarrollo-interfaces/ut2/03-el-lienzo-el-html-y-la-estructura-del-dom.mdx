Seguimos nuestro viaje.  
Ya diferenciamos entre pensar de forma **imperativa** y **declarativa**.  
Ahora toca conocer a fondo la herramienta declarativa que sostiene toda la web: **HTML**.  

Imagina que HTML es un set de **piezas de construcción**.  
Cada pieza (`<h1>`, `<p>`, `<div>`, `<button>`) ya está fabricada.  
Tu tarea no es inventarlas, sino **combinarlas con sentido** para construir una interfaz.  

La magia es que, al encajar esas piezas, el navegador sabe **qué significan** y las convierte en una estructura viva: el **DOM (Document Object Model)**.

---

## HTML: Lenguaje Declarativo para Interfaces

HTML no es un lenguaje para calcular o procesar datos.  
Es un lenguaje diseñado para **estructurar interfaces de usuario** con significado semántico.

```html
<h1>Mi Título</h1>
```

Con esto declaras: “Esto es un encabezado principal”.

No dices “píntalo grande y en negrita”: eso ya lo sabe el navegador.

> HTML es un lenguaje declarativo porque describes **qué es** cada pieza, no cómo dibujarla.

---

## La Jerarquía Padre-Hijo: El Árbol DOM

Cuando el navegador interpreta tu HTML, lo transforma en un **árbol de nodos en memoria**: el **DOM**.

```html
<main>
  <section>
    <h2>Título de la sección</h2>
    <p>Un párrafo...</p>
  </section>
</main>
```

- `<main>` es el nodo raíz de tu contenido.
- `<section>` es hijo de `<main>`.
- `<h2>` y `<p>` son hijos de `<section>`.

Esta jerarquía es **fundamental**, porque es la que usaremos luego con JavaScript para seleccionar, modificar y añadir elementos.

---

## Buenas Prácticas: Cimientos Sólidos

Escribir HTML es fácil. Escribir **buen HTML** es lo que distingue a un desarrollador profesional.

- **Semántica primero**: Usa etiquetas con su propósito real (`<nav>` para navegación, `<main>` para contenido principal, `<article>` o `<section>` para bloques independientes).
- **Accesibilidad (a11y)**: Usa `alt` en imágenes (`<img>`) para describirlas y asocia `<label>` con `<input>` para mejorar la experiencia.
- **Validación y limpieza**: Pasa tu código por un validador como el de W3C e indenta y organiza para que la jerarquía se entienda de un vistazo.

> Un HTML semántico mejora la accesibilidad, el SEO y la mantenibilidad.

---

## Formularios: El Corazón de la Interacción

Los formularios son la puerta de entrada para que el usuario interactúe con la web.

- **El dúo `<label>` + `<input>`**: siempre juntos.

  ```html
  <label for="nombre">Nombre:</label>
  <input id="nombre" type="text" required />
  ```

- **Validación declarativa (HTML5)**: `required`, `minlength`, `maxlength`, `pattern`...

Ejemplo: un email válido.

```html
<input 
  type="email" 
  id="correo" 
  required 
  pattern="^[^@]+@[^@]+\\.[a-zA-Z]{2,}$" 
/>
```

---

## Atributos: Metadatos y Ganchos

Los atributos son información adicional para cada etiqueta.

- **`id`**: único, como el DNI del elemento. Útil para JS (`getElementById`).
- **`class`**: agrupa elementos. Perfecta para CSS o `querySelectorAll`.
- **`data-*`**: para guardar metadatos personalizados.

```html
<button data-user-id="42" data-action="delete">
  Eliminar Usuario
</button>
```

En JS se accede con `element.dataset.userId` o `element.dataset.action`.
