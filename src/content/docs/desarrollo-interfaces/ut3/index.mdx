En esta Unidad de Trabajo (UT3) daremos un paso crucial hacia el desarrollo de interfaces modernas y complejas. Si bien en la UT2 exploramos la manipulación del DOM con JavaScript puro, entendimos que para aplicaciones de gran escala, necesitamos herramientas más potentes y estructuradas: los **frameworks**.

Ha llegado el momento de sumergirnos en **Angular**, un framework robusto y completo, respaldado por Google, que nos permitirá construir aplicaciones web de una sola página (SPA) de manera eficiente y escalable. Angular nos proporciona una arquitectura sólida, un conjunto de herramientas integradas y una filosofía de desarrollo que facilita la creación de interfaces dinámicas e interactivas.

Imagina que si HTML es el esqueleto y CSS la piel, JavaScript puro es el sistema nervioso que da movimiento. Con Angular, no solo tenemos un sistema nervioso, sino un **cerebro completo** que organiza, gestiona y optimiza todas las funciones, permitiéndonos construir organismos digitales mucho más sofisticados.

---

## Objetivos de la Unidad

Al finalizar esta unidad serás capaz de:

-   Comprender la necesidad y las ventajas de utilizar frameworks en el desarrollo web moderno.
-   Conocer los fundamentos de Angular, su arquitectura basada en componentes y módulos.
-   Ser capaz de crear, configurar y ejecutar un proyecto básico con Angular CLI.
-   Identificar los componentes principales de una aplicación Angular (componentes, servicios, rutas).
-   Desarrollar componentes básicos, entender su ciclo de vida y la comunicación entre ellos.
-   Implementar navegación entre diferentes vistas utilizando el sistema de rutas de Angular.
-   Manejar formularios y realizar validaciones en aplicaciones Angular.
-   Consumir datos de APIs externas utilizando el módulo `HttpClient` de Angular.

---

## Contenidos Clave

1.  **Introducción a Angular**: Qué es, por qué usarlo y su posicionamiento en el ecosistema de frameworks.
2.  **Tu Primer Proyecto Angular**: Creación de un proyecto con Angular CLI, estructura de archivos y ejecución.
3.  **Componentes en Angular**: Creación, estructura (HTML, TypeScript, CSS) y comunicación entre componentes.
4.  **Servicios e Inyección de Dependencias**: Gestión de la lógica de negocio y el acceso a datos de forma modular.
5.  **Rutas y Navegación**: Configuración del sistema de rutas para crear Single Page Applications (SPAs).
6.  **Formularios en Angular**: Implementación de formularios reactivos y basados en plantillas, y su validación.
7.  **Comunicación con APIs (HttpClient)**: Realización de peticiones HTTP para interactuar con servicios backend.
