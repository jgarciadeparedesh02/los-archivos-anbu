## Introducción

En esta práctica aplicarás los conceptos de diseño de interfaces y prototipado para crear un **prototipo interactivo de alta fidelidad** en Figma. Diseñarás las pantallas de autenticación de una aplicación web (inicio de sesión y registro), prestando especial atención a la **usabilidad**, la **retroalimentación al usuario** y la **navegación entre pantallas**.

Imagina que formas parte del equipo de diseño de una aplicación web ficticia (ej. una tienda online o red social) y tu tarea es construir el flujo de autenticación inicial.

## Objetivos

* Familiarizarte con las herramientas básicas de diseño y prototipado de Figma.
* Diseñar componentes de interfaz de usuario comunes: formularios, botones y barra de navegación.
* Implementar flujos de navegación básicos entre pantallas.
* Aplicar principios de usabilidad, validación de datos y retroalimentación clara al usuario.
* Mantener coherencia visual mediante paleta de colores, tipografía y estilo de componentes.
* Incluir opciones de inicio de sesión y registro mediante redes sociales.

## Enunciado de la práctica

Crea en Figma un **prototipo interactivo** que muestre el flujo de autenticación de una aplicación web. El prototipo debe incluir al menos lo siguiente:

### 1. Estructura general

* **Barra de navegación (Navbar):** presente en ambas pantallas, con enlaces a secciones ficticias:
  * Inicio
  * Perfil
  * Tienda
  * Contacto
* **Consistencia visual:** colores, tipografía y componentes deben ser coherentes en todas las pantallas.

### 2. Pantalla de inicio de sesión (Login)

Esta es la pantalla principal del flujo.

1. **Campos del formulario:**
   * Email o nombre de usuario.
   * Contraseña.
2. **Funcionalidades y enlaces:**
   * Enlace para **“Recordar contraseña”**.
   * Checkbox o interruptor para **“Mantener sesión iniciada”**.
   * Botón visible para **“Iniciar sesión”**.
   * Enlace hacia la pantalla de registro: **“¿No tienes una cuenta? Regístrate”**.
   * Opciones de **inicio de sesión con redes sociales** (ej. Google, Facebook, Twitter).
3. **Validaciones y retroalimentación:**
   * Mostrar un estado de error si los datos son incorrectos (ej. email sin `@`, contraseña demasiado corta).
   * Mensajes de error claros y específicos.

### 3. Pantalla de registro

Permite a los nuevos usuarios crear una cuenta.

1. **Campos del formulario:**
   * Nombre completo.
   * Dirección de email.
   * Contraseña.
   * Confirmar contraseña.
2. **Funcionalidades y enlaces:**
   * Botón para **“Crear cuenta”**.
   * Enlace para volver al login: **“¿Ya tienes una cuenta? Inicia sesión”**.
   * Opciones de **registro con redes sociales** (ej. Google, Facebook, Twitter).
3. **Validaciones y retroalimentación:**
   * Errores si:
     * Las contraseñas no coinciden.
     * El email no tiene formato válido.
     * Algún campo obligatorio está vacío.
   * Mensajes claros y específicos.
   * Estado de éxito tras el registro (ej. “Tu cuenta ha sido creada”).

### 4. Interactividad del prototipo

* El enlace **“Regístrate”** en la pantalla de Login debe navegar al Registro.
* El enlace **“Inicia sesión”** en el Registro debe volver al Login.
* Los botones deben simular interacción con retroalimentación visual (ej. hover, disabled).
* Las opciones de acceso mediante redes sociales deben simular navegación (aunque no sea funcional).

## Sugerencias de trabajo

* **Componentes:** crea componentes en Figma para botones, campos de texto, navbar y accesos sociales, de modo que puedas reutilizarlos fácilmente.
* **Auto Layout:** úsalo para formularios y grupos de elementos, facilitando consistencia y adaptabilidad.
* **Inspiración:** observa ejemplos de login/registro en apps conocidas, pero desarrolla un diseño original.
* **Feedback visual:** añade estados (normal, hover, error, éxito, disabled) para botones y campos.
* **Accesibilidad:** asegúrate de buen contraste de colores, fuentes legibles y etiquetas claras.

* Link al prototipo interactivo en Figma.
* Capturas de las pantallas principales (Login y Registro).

## Criterios de evaluación

* **Funcionalidad del flujo** (login ↔ registro con enlaces interactivos).
* **Consistencia visual y uso de componentes.**
* **Claridad y usabilidad** de formularios, mensajes de error y accesos sociales.
* **Creatividad y presentación visual.**
* **Aplicación de buenas prácticas** (feedback, accesibilidad básica, orden).
