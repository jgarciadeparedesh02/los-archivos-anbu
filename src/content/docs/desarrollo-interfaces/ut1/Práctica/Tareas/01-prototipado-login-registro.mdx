## Introducción

En esta práctica, aplicaremos los conceptos de diseño de interfaces y prototipado para crear un prototipo interactivo de alta fidelidad de una aplicación web utilizando Figma. El objetivo es diseñar las pantallas de autenticación (inicio de sesión y registro), prestando especial atención a la usabilidad, la retroalimentación al usuario y la navegación.

## Objetivos

- Familiarizarse con las herramientas básicas de diseño y prototipado de Figma.
- Diseñar componentes de interfaz de usuario comunes como formularios, botones y barras de navegación.
- Implementar flujos de navegación básicos entre diferentes pantallas.
- Aplicar principios de usabilidad para la validación de datos y la retroalimentación al usuario.

## Enunciado de la Práctica

Se debe crear un prototipo en Figma que simule el flujo de autenticación de una aplicación web. El prototipo debe incluir, como mínimo, los siguientes elementos:

### 1. Estructura General

- **Barra de Navegación (Navbar):** Debe estar presente en la parte superior de ambas pantallas. Incluirá enlaces a secciones hipotéticas de la web como:
    - Inicio
    - Perfil
    - Tienda
    - Contacto
- **Consistencia Visual:** Ambas pantallas deben compartir una paleta de colores, tipografía y estilo de componentes coherente.

### 2. Pantalla de Inicio de Sesión (Login)

Esta será la pantalla principal del flujo.

- **Campos del formulario:**
    - Email o nombre de usuario.
    - Contraseña.
- **Funcionalidades y Enlaces:**
    - Un enlace para **"Recordar contraseña"**.
    - Un checkbox o interruptor para **"Mantener sesión iniciada"**.
    - Un botón bien visible para **"Iniciar Sesión"**.
    - Un enlace a la pantalla de registro, como por ejemplo: **"¿No tienes una cuenta? Regístrate"**.
- **Validación de Errores:**
    - Se debe mostrar un estado de error si los datos introducidos no son correctos (p. ej., el campo de email no contiene un `@` o la contraseña es demasiado corta). El mensaje de error debe ser claro y específico.

### 3. Pantalla de Registro

Esta pantalla permitirá a los nuevos usuarios crear una cuenta.

- **Campos del formulario:**
    - Nombre completo.
    - Dirección de email.
    - Contraseña.
    - Confirmar contraseña.
- **Funcionalidades y Enlaces:**
    - Un botón para **"Crear Cuenta"**.
    - Un enlace para volver a la pantalla de login, como: **"¿Ya tienes una cuenta? Inicia Sesión"**.
- **Validación de Errores:**
    - Mostrar mensajes de error para casos como:
        - Las contraseñas no coinciden.
        - El formato del email no es válido.
        - Algún campo obligatorio está vacío.

### 4. Interactividad del Prototipo

- El enlace "Regístrate" en la pantalla de Login debe navegar a la pantalla de Registro.
- El enlace "Inicia Sesión" en la pantalla de Registro debe navegar de vuelta a la pantalla de Login.

## Sugerencias

- **Uso de Componentes:** Crea componentes en Figma para elementos reutilizables como botones, campos de texto y la barra de navegación. Esto facilitará la consistencia y la edición del diseño.
- **Auto Layout:** Utiliza el "Auto Layout" de Figma para crear formularios y componentes que se adapten dinámicamente al contenido.
- **Inspiración:** Busca ejemplos de pantallas de login y registro en sitios web populares para inspirarte, pero crea un diseño original.
- **Feedback Visual:** Asegúrate de que los botones y campos interactivos tengan estados visuales claros (normal, hover, disabled).

## Criterios de Evaluación

- **Cumplimiento de Requisitos:** Se han incluido todas las pantallas, campos, enlaces y validaciones solicitadas.
- **Calidad del Diseño:** El diseño es limpio, coherente y sigue principios básicos de usabilidad y jerarquía visual.
- **Funcionalidad del Prototipo:** La navegación entre las pantallas de login y registro funciona correctamente.
- **Claridad:** Los mensajes de error y las instrucciones son fáciles de entender para el usuario.
