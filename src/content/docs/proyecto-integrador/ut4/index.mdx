## Objetivo

¡Manos a la obra! Esta es la fase de construcción, donde el diseño y la planificación se convierten en un producto funcional. Aplicando una metodología ágil, desarrollarás la aplicación de forma iterativa, construyendo y entregando funcionalidades en ciclos cortos (sprints). El objetivo no es tener todo perfecto desde el principio, sino construir, probar y mejorar de manera continua.

## Tareas a realizar

1.  **Planificación del Desarrollo (Product Backlog)**:
    - Desglosa todas las historias de usuario y requisitos funcionales de la UT2 en tareas técnicas más pequeñas. Esto formará tu **Product Backlog**.
    - Prioriza las tareas. ¿Qué es lo más importante para tener un Producto Mínimo Viable (MVP)?

2.  **Ciclos de Desarrollo (Sprints)**:
    - Organiza tu trabajo en Sprints de 1 a 2 semanas.
    - Al inicio de cada Sprint, selecciona un conjunto de tareas del backlog para completar (**Sprint Backlog**).
    - Al final de cada Sprint, deberías tener una nueva versión funcional de la aplicación que incluya las nuevas características.

3.  **Desarrollo Frontend y/o Backend**:
    - Implementa las funcionalidades planificadas siguiendo la arquitectura y las convenciones definidas en la UT3.
    - Crea componentes reutilizables, gestiona el estado de la aplicación y conecta el frontend con el backend o los servicios externos si es necesario.

4.  **Integración Continua**:
    - Utiliza un sistema de control de versiones (Git) de forma profesional:
        - Trabaja en ramas separadas para cada nueva funcionalidad (`feature-branches`).
        - Realiza Pull Requests para integrar los cambios en la rama principal (`main` o `develop`).
        - Asegúrate de que el código nuevo no rompa la funcionalidad existente.

## Entregable

El entregable de esta fase es el propio **código fuente de la aplicación**, gestionado a través de un repositorio de Git. El progreso se medirá por las funcionalidades completadas y la calidad del código. Se valorará la organización del repositorio y el seguimiento de una metodología de trabajo ágil.
