## Objetivo

Un buen producto no solo debe funcionar, sino que debe ser fiable, robusto y libre de errores. En esta fase, te centrarás en asegurar la calidad de tu aplicación a través de un plan de pruebas sistemático. Además, aprenderás a utilizar herramientas de gestión para hacer un seguimiento del progreso y controlar el alcance del proyecto.

## Tareas a realizar

1.  **Plan de Pruebas (Testing)**:
    - Define una estrategia de testing. ¿Qué tipos de pruebas realizarás?
    - **Pruebas Unitarias**: Escribe tests para verificar que las funciones y componentes individuales funcionan como se espera.
    - **Pruebas de Integración**: Comprueba que varios componentes juntos interactúan correctamente.
    - **Pruebas End-to-End (E2E)**: Automatiza flujos de usuario completos para simular el comportamiento de una persona real en la aplicación (ej: con Cypress o Playwright).
    - **Pruebas Manuales**: Define un conjunto de casos de prueba para ejecutarlos manualmente y detectar errores de usabilidad o visuales.

2.  **Gestión de Incidencias (Bug Tracking)**:
    - Utiliza una herramienta (como GitHub Issues, Trello, Jira) para registrar, priorizar y hacer seguimiento de los errores que encuentres durante las pruebas.

3.  **Seguimiento del Proyecto**:
    - Mantén actualizado un tablero Kanban (en Trello, GitHub Projects, etc.) para visualizar el estado de todas las tareas (Pendiente, En Progreso, Hecho).
    - Realiza reuniones de seguimiento periódicas (aunque sea contigo mismo) para evaluar el progreso, identificar bloqueos y ajustar la planificación si es necesario.

4.  **Control de Calidad del Código**:
    - Ejecuta las herramientas de linting y formateo para asegurar que todo el código sigue las mismas convenciones.
    - Realiza revisiones de código (Code Reviews) en los Pull Requests para mejorar la calidad y compartir conocimiento.

## Entregable

Elaborarás un **documento de plan de pruebas** que describa tu estrategia y casos de prueba. Además, el **historial de incidencias** y el **tablero de seguimiento** del proyecto servirán como evidencia de una gestión de proyecto y un control de calidad adecuados.
